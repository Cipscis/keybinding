Keybinding documentation

Allows you to bind and unbind events to individual keypresses or sequences of keypresses. Intended for implementing keyboard shortcuts.

bind

bind(key, fn, allowInInput, requireCtrl)

Binds a callback to a specified keypress.

Arguments

key

A string of the name of the key to bind to. For a list, see Key Values on MDN.

Some key names also have aliases:

Space
'spacebar'
Up arrow
'up'
Right arrow
'right'
Down arrow
'down'
Left arrow
'left'

fn

The function to bind to the keypress.

allowInInput

Unless this or `requireCtrl` are set to `true`, the keypress will not be checked if the keyboard focus is currently inside a form element.

requireCtrl

If set to `true`, the bound function will only be fired if the Ctrl key is pressed.

unbind

unbind(key, fn)

Unbinds a function from a specified keypress.

Arguments

key

A string of the name of the key to bind to.

fn

The function to unbind to the keypress.

bindSequence

bindSequence(...keys, fn)

Binds a function to a sequence of keypresses. They must be pressed in order, with no other keypresses between them, before the function will be called.

Arguments

keys

The names of the keys, in order, that should be pressed in order to fire the bound function.

fn

The function to bind to the sequence of keypresses.

unbindSequence

unbindSequence(...keys, fn)

Unbinds a function from a sequence of keypresses.

Arguments

keys

The names of the keys, in order, that the function is bound to.

argB

THe function to unbind from the sequence of keypresses.

Examples

Try the following keyboard shortcuts/sequences:

  • a
  • Ctrl + m (will work while focus in input)
  • Konami code (up up down down left right left right b a enter)

Keyboard shortcuts won't fire when your focus is in an input, unless it's specified that it should.

Keep in mind that only some browser shortcuts (e.g. Ctrl + s) can have their default action prevented with event.preventDefault(). If you ever do this, be sure to test it in multiple browsers.
